> 主题：流形单纯复形 + 网格拓扑数据结构（邻接表 / 关联矩阵 / 半边）

## 一、流形：什么是"好空间"

### 1. 动机

- **流形（Manifold）** 是微分几何中**"好空间"**的模型——一种简化假设，使分析与计算变得容易。
- 类比：参数化曲线中假设"弧长参数化"避免奇异性；流形帮助我们忽略"怪异"空间。
- **关键问题**：什么是"好"？答案要从**局部**而非全局判断。

### 2. 直观定义：局部像欧氏空间

> **核心思想**：在流形上的**每一点**附近的邻域里，空间看起来都像**欧几里得空间**（二维流形像 $\mathbb{R}^2$，三维流形像 $\mathbb{R}^3$）。

- **关键操作**：在曲面一点上**放大（zoom in）**，直到它看起来像一个可以画上"横平竖直网格"的**小平面片**。
- **直观比喻**：地球表面整体弯曲，但站在操场上时局部完全平坦，可以用 $xy$ 坐标描述位置。
- **判定**：放大该点的邻域：
  - **流形**：邻域可平滑映射到平面网格，无撕裂、无奇异点。
  - **非流形**：邻域呈现两条线交叉、断裂、或无法铺展（如锥点、沙漏腰部）。

### 3. 形式化定义（连续情形）

对于点 $p \in M$，若存在一个**同胚** $\phi: B \to U$，将 $\mathbb{R}^k$ 中的开球 $B$ 映射到 $p$ 的开邻域 $U$，则 $M$ 在 $p$ 处是 $k$ 维流形。

- **同胚（Homeomorphism）**：连续变形（不撕裂、不粘连）下的等价。
- **奇异点（Singularity）**：无法局部坐标化的点，如沙漏腰部（无论怎么放大都是两个锥体相交）。

## 二、判定单纯复形是否为流形

### 1. 用 Link 的精确定义

> 一个 $k$ 维单纯复形 $K$ 是无边界的流形，当且仅当对 $K$ 中每个顶点 $v$，**$\text{Lk}(v)$ 与 $(k-1)$ 维球面 $\mathbb{S}^{k-1}$ 拓扑同胚**。

| 维度 | Link 应是 | 拓扑解释 |
| :---: | :--- | :--- |
| $k=1$（线段网格） | 0 维球面 = **两个孤立点** | 顶点恰好两个邻居 |
| $k=2$（三角网格） | 1 维球面 = **单一闭合环** | 围绕顶点的一圈边构成单环 |
| $k=3$（四面体网格） | 2 维球面 = **拓扑球面**（如二十面体） | 围绕顶点的一圈四面体的面构成封闭多面体 |

**记忆口诀**：**"二维看环，三维看球，一维看两点。"**

### 2. 经典反例

| 形状 | 中心点的 Link | 是否流形 |
| :--- | :--- | :---: |
| 二十面体的顶点 | 5 边的闭环 | ✅ |
| 马鞍点 | 闭环 | ✅ |
| **沙漏点（Pinch Point）** | **两个分离的环** | ❌ |
| **五角星中心** | **五个孤立顶点（不构成环）** | ❌ |
| 三个不连接的三角形 | — | ❌（不连通） |

**沙漏分析**：手指绕中心顶点周围走，先绕完第一个锥体，然后必须**跳跃**穿过顶点才能开始绕第二个——无法**连续**走完所有相邻三角形。

### 3. 计算可行性

| 维度 | 检查方法 | 复杂度 |
| :--- | :--- | :--- |
| 一维（图） | 检查每顶点是否有 2 个邻居（Y 形交叉点为 3 个邻居 → 非流形） | 简单 |
| 二维 | 检查每顶点的 Link 是否为单环 | 简单 |
| 三维 | 检查每顶点的 Link 是否为拓扑球面 | **NP 问题**——识别三维球体目前**无人知道**是否有多项式时间算法 |

### 4. 欧拉公式（二维流形的简易必要条件）

$$V - E + F = 2$$

- 这是二维封闭流形（如球面）必须满足的关系。
- **必要条件**，不充分。
- 三维情形可对每顶点的 Link 检查此公式判别是否是球面（作为快速检查）。

### 5. 三角网格流形：实用判定法则

对游戏引擎/渲染中实际使用的三角网格，有**直观且易操作**的法则：

#### 边的判定

- **内部边**：恰好被 **2** 个三角形共享。
  - 反例：一条边被 5 个三角形共享（**非流形边**）。
- **边界边**：恰好被 **1** 个三角形共享。

#### 顶点的判定

- **内部顶点**：周围所有三角形构成**单一完整的环**（链接条件）。
  - 反例：两个锥体共享一个顶点（**非流形顶点**）。
- **边界顶点**：周围所有三角形构成**单一扇形**（不是多扇形）。

**速查表**：

| 类别 | 条件 |
| :--- | :--- |
| 内部边 | 正好 2 个三角形 |
| 边界边 | 正好 1 个三角形 |
| 内部顶点 | 单一三角形环 |
| 边界顶点 | 单一三角形扇 |

任何违背 → 非流形。

## 三、为什么追求流形网格

### 1. 二维图像类比

像素理论上可以是六边形或不规则形状，但事实标准是**规则方形像素网格**：
1. **功能足够**：能表示任何图像。
2. **结构简单可预测**：每像素固定 4 个邻居（上下左右）。
3. **算法设计与数据存储简单高效**。

### 2. 流形网格的价值

类比推广：流形网格 ≈ 规则像素网格的高维版。

- **可预测的邻域结构**：
  - 每条**边**始终只有左、右两个三角形。
  - 每个**顶点**的环绕三角形构成单一封闭环。
- **足够好的表达能力 + 极其简单规律的结构** → 设计**高效拓扑数据结构**的基础。

## 四、网格拓扑数据结构

### 1. 拓扑数据结构 vs 几何数据

- **拓扑数据结构（Topological Data Structure）**：编码网格**连通性**——谁与谁相连。
- **几何数据**：顶点的具体空间位置。
- 二者分离存储，使邻域查询更高效。

### 2. 邻接表（Adjacency List）

- **思想**：只存最高维单纯形（三角网格只存三角形的三个顶点索引）。
- **示例（空心四面体）**：`[0,1,2], [0,3,2], [3,1,2], [0,1,3]`
- **优势**：极其简单，存储成本低。
- **致命缺陷**：
  - 无法直接获取边/顶点列表（需手动从三角形列表去重提取）。
  - **邻域查询代价极高**：要找顶点 $v$ 的邻边，必须遍历**全部**三角形。在十亿级网格上每次查询要十亿次操作——不可接受。
- **结论**：仅适合作为**文件存储格式**，不适合实时邻域处理。任何有经验的渲染工程师会立刻把它转换成更高级的结构（如半边）。

### 3. 关联矩阵（Incidence Matrix）

#### 3.1 定义

设 $K$ 是单纯复形，$n_k$ 为 $k$ 维单纯形数量。第 $k$ 个关联矩阵 $\mathbf{E}_k$ 是 $n_{k+1} \times n_k$ 矩阵：

$$(\mathbf{E}_k)_{ij} = \begin{cases} 1, & \text{第 } j \text{ 个 } k \text{-单纯形} \subseteq \text{第 } i \text{ 个 } (k+1)\text{-单纯形} \\ 0, & \text{否则} \end{cases}$$

| 矩阵 | 行 | 列 | 每行 1 数 | 每列 1 数（流形条件下） |
| :--- | :--- | :--- | :---: | :---: |
| $\mathbf{E}_0$ | 边 | 顶点 | 2（每边两端点） | 顶点的度（视拓扑） |
| $\mathbf{E}_1$ | 三角形 | 边 | 3（每三角形三条边） | **2**（每条边恰好属于 2 个三角形） |

> **空心四面体**：$\mathbf{E}_0$ 是 $6 \times 4$（6 条边、4 个顶点），$\mathbf{E}_1$ 是 $4 \times 6$（4 个面、6 条边）。

#### 3.2 查询：从图遍历到线性代数

- **顶点 $v$ 的邻边**：查 $\mathbf{E}_0$ 第 $v$ 列中值为 1 的行索引。
- **边 $e$ 的邻接三角形**：查 $\mathbf{E}_1$ 第 $e$ 列中值为 1 的行索引。
- **顶点 $v$ 的关联三角形（组合查询）**：本质上是 $\mathbf{E}_1 \cdot \mathbf{E}_0$ 的第 $(i, v)$ 元素，表示从顶点 $v$ 经一条边到达三角形 $i$ 的路径数。

### 4. 稀疏性：关联矩阵的真正威力

#### 4.1 为什么矩阵稀疏

- 大多数网格元素只与少量相邻元素关联。
- **流形网格关键事实**：$\mathbf{E}_1$ 的**每列只有 2 个 1**——每条边恰好属于 2 个三角形。

#### 4.2 直接稠密存储的瓶颈

要找"顶点 3 的邻边"扫描 $\mathbf{E}_0$ 的第 3 列：列长十亿级 → $O(n_k)$ → 没比邻接表好。

#### 4.3 稀疏存储方案对比

| 方案 | 思想 | 优点 | 缺点 | 用途 |
| :--- | :--- | :--- | :--- | :--- |
| **关联数组** | $(行, 列) \mapsto$ 值，哈希表/树实现 | 概念简单；插入/查询方便；天然不存零 | 不适合矩阵运算（如矩阵-向量乘法低效） | 构建矩阵的中间结构 |
| **行链表数组** | 每行一个链表存非零元 | 概念简单 | 访问慢，缓存不友好 | 较少使用 |
| **CSC（压缩列）** | 三个数组：值数组、行索引数组、列指针数组 | 极致紧凑；高效矩阵运算；缓存友好 | 实现略复杂 | **实际计算最常用** |

**CSC 示例**：
- 值数组 `[4, 2, 7, 3]`
- 行索引 `[0, 2, 1, 0]`
- 列指针 `[0, 2, 4]`（列 0 有 2 个非零元，索引 0-1；列 1 有 2 个非零元，索引 2-3）

#### 4.4 复杂度飞跃

- **存储**：$O(n_{k+1} \cdot n_k) \to O(\text{非零元数})$，对流形网格通常与元素数线性相关。
- **查询**：边的两个面只需读两个面索引 → **$O(1)$**。

> **核心结论**：关联矩阵的优势**不来自矩阵本身**，而来自其**天然稀疏性**。利用 CSC 等稀疏格式才能达到效率。

## 五、带符号关联矩阵：引入方向

### 1. 定义

在普通关联矩阵的 0/1 基础上，引入 $+1$ 和 $-1$ 表示**相对方向**：

- **$+1$**：方向一致
- **$-1$**：方向相反
- **$0$**：不关联

### 2. $\mathbf{E}_0$（顶点-边）

设边 $e$ 的方向为 $v_a \to v_b$：
- 行 $e$、列 $v_a$（起点）：$-1$
- 行 $e$、列 $v_b$（终点）：$+1$

> 直觉：起点是 $-1$，终点是 $+1$。

### 3. $\mathbf{E}_1$（边-三角形）

行代表三角形，列代表有向边：

- **$+1$**：三角形环绕方向与该边方向**一致**。
- **$-1$**：三角形环绕方向与该边方向**相反**。

**示例**：三角形 $t_1$ 的顶点顺序 $v_1 \to v_2 \to v_3$（顺时针），其中边 $e_2$ 方向为 $v_2 \to v_3$ 一致 → $\mathbf{E}_1[t_1, e_2] = +1$。

### 4. 重要前瞻

带符号关联矩阵在 DEC（**离散外微积分**）中具有**远比拓扑表层更深的数学含义**——是离散梯度、散度、旋度的基础。

## 六、半边数据结构（Half-Edge）

> 本课程中处理网格几乎**唯一**会用到的数据结构。

### 1. 核心思想

把每条**无向边**分割为两条**方向相反**的**有向半边**。半边像"胶水"把所有元素粘起来。

### 2. 半边对象包含的信息

| 字段 | 含义 |
| :--- | :--- |
| **Twin** | 同属一条边但方向相反的另一条半边 |
| **Next** | 当前面内顺序的下一条半边 |
| **Vertex (From)** | 这条半边出发的顶点 |
| **Edge** | 所属的原始无向边 |
| **Face** | 所属的三角形面 |

### 3. 其他元素的"最小知识"原则

- **边（Edge）**：只需记录它的**一条**半边；另一条用 Twin 找。
- **面（Face）**：只需记录它的**一条**半边；其它用 Next 反复迭代获取。

### 4. 网格遍历模式

| 操作 | 实现 |
| :--- | :--- |
| **绕一个面循环** | 反复 `Next` |
| **跨越边到相邻面** | `Twin` |
| **绕一个顶点循环** | 反复 `Twin` → `Next` → `Twin` → `Next` ... |

### 5. 抽象代数视角

半边数据结构等价于一个**纯代数结构**——两个映射作用于一个集合上。

#### 5.1 形式化定义

| 元素 | 含义 |
| :--- | :--- |
| **集合 $H$** | 所有半边；$\lvert H\rvert$ **必为偶数**（半边成对出现） |
| **映射 $\rho$（即 next）** | $\rho: H \to H$，是**置换**（双射） |
| **映射 $\eta$（即 twin）** | $\eta: H \to H$，是**对合且无不动点**的置换 |

#### 5.2 $\eta$ 的两条性质

- **对合性**：$\eta(\eta(h)) = h\quad \forall h \in H$
  - 白话："我的 twin 的 twin 就是我"。
- **无不动点**：$\eta(h) \neq h\quad \forall h \in H$
  - 白话："我的 twin 不可能是我自己"。

#### 5.3 几何对象的代数构造（轨道法）

| 几何对象 | 代数构造 |
| :--- | :--- |
| 半边 | $H$ 中的元素 |
| **边** | $\eta$ 的**轨道**——反复应用 $\eta$ 直到回到起点 |
| **面** | $\rho$ 的**轨道**——反复应用 $\rho$ 直到回到起点 |
| **顶点** | $\eta \circ \rho$ 的**轨道** |

#### 5.4 简化约定

- 约定半边 $1\leftrightarrow 2$、$3\leftrightarrow 4$、$5\leftrightarrow 6$ ... 互为 twin。
- 这样 $\eta$ **隐式定义**，无需显式存储。

#### 5.5 颠覆性洞察

> **任何偶数大小集合上的任何置换 $\rho$ 描述一个流形网格。**

只要取偶数大小的集合 + 任意置换 $\rho$，追踪轨道即可得到合法流形网格。

#### 5.6 代数构造示例

10 个半边 $H_0, \dots, H_9$，定义：

- $\rho$：$H_0 \to H_1 \to H_2 \to H_0$，$H_3 \to H_4 \to H_5 \to H_3$，$H_6 \to H_7 \to H_8 \to H_9 \to H_6$
- $\eta$：$H_0 \leftrightarrow H_3$，$H_1 \leftrightarrow H_2$，等等

轨道追踪：

1. **边**：$\eta(H_0) = H_3,\ \eta(H_3) = H_0$ → 一条边
2. **面**：$\rho$ 三步循环 $H_0, H_1, H_2$ → 一个三角形
3. **顶点**：$\eta \circ \rho$ 的轨道

**结果**：两个三角形面 + 一个由 4 半边构成的外部环。

## 七、关键术语速查

| 术语 | 含义 |
| :--- | :--- |
| 流形 | 局部与欧氏空间 $\mathbb{R}^k$ 同胚的空间 |
| 同胚 | 连续变形（不撕裂、不粘连）下的等价 |
| 奇异点 | 无法局部坐标化的点（如沙漏腰部） |
| Link 流形条件 | 每顶点的 Link $\cong \mathbb{S}^{k-1}$ |
| 三角网格流形判定 | 内部边在 2 面、边界边在 1 面、内部顶点为单环、边界顶点为单扇 |
| 邻接表 | 仅存最高维单纯形 → 邻域查询慢 |
| 关联矩阵 $\mathbf{E}_k$ | $(k+1)$-单纯形 × $k$-单纯形的关联 0/1 矩阵 |
| 带符号关联矩阵 | 在 0/1 基础上引入 $\pm 1$ 表示方向一致/相反 |
| 稀疏矩阵（CSC） | 三数组紧凑存储，将查询降至 $O(1)$ |
| 半边 | 每条无向边的两条有向版本 |
| Twin | 同条边的反方向半边（$\eta$） |
| Next | 同一面内顺序下的半边（$\rho$） |
| 对合 + 无不动点 | $\eta^2 = \mathrm{id}$ 且 $\eta(h) \neq h$ |
| 半边轨道 | 边 = $\eta$ 轨道；面 = $\rho$ 轨道；顶点 = $\eta \circ \rho$ 轨道 |
